JDK 1.5新特性

1. 自动装箱和拆箱

自动装箱：基本类型自动封装成包装类的过程，如 int -> Integer。

自动拆箱：封装类自动转成基本类型的过载，如 Double -> double。

2. 枚举

把集合里的对象元素一个一个提取出来。枚举类型使代码更具可读性，理解清晰，易于维护。枚举类型是强类型的，从而保证了系统的安全性。而类的静态字段不具有枚举的简单性和类型安全性。

3. 静态导入

通过使用import static，就可以不用指定类名而可以直接使用静态变量或静态方法。

import和import static的区别是前者一般导入的是类文件如imort java.util.Scanner，后者一般是导入静态方法，如import static java.lang.System.out。

4. 可变参数(Varargs)

可以传入0到n个参数，使用方法为如：methodName(int arg1, int arg2, String … strs)，可变参数是放在最后面的。

5. 内省(Introspector)

是Java对Bean类属性，事件的一种缺省处理方法。如类中有一个属性name，那么可以通过getName来获取其值，通过setName来设置其值。

6. 泛型(Generic)

C++通过模板技术可以指定集合的元素类型，而Java在1.5之前一直没有相应的功能。一个集合可以放任何类型的对象，那么从集合里获取对象的时候我们必须要强制类型转换。引入泛型后，它允许指定集合中元素的类型，这样可以得到强类型在编译时刻进行类型检查的好处。

7. For-Each循环

For-Each循环 简化了集合的遍历。
